⚠️ Lecture search unavailable
The active model does not support tool calling, so it cannot search your uploaded lecture slides. Students will still get answers, but they won't be grounded in your course material and no slide citations will appear.
To enable lecture search, switch to a tool-capable model:
  • Local: qwen3.5:4b
  • Cloud: gpt-oss:120b-cloud, qwen3.5:397b-cloud, deepseek-v3.1:671b-cloud
Custom Mode — guardrails come from the model
Custom Mode sends no system prompt — no Professor Tux persona, no identity guard, no safety rules, no teaching style. The model receives only the conversation history and the student's message.
Whatever guardrails you need (refusing offensive use, hiding the underlying model name, scope to cybersecurity, output style) must be built into the model itself — either via fine-tuning, an Ollama Modelfile SYSTEM directive, or a model that already enforces them internally.
Use this mode when you want to evaluate the raw model behaviour, or when the active model already carries its own teaching persona.
CTF Mode — tool-capable model recommended
CTF Mode can use Ollama's cloud web_search API and tool calling. Live web search requires an Ollama API key; local chat still uses your configured Ollama endpoint.
Before switching, make sure a tool-capable model is active. Cloud models are recommended for CTF work, for example:
  • gpt-oss:120b-cloud
  • qwen3.5:397b-cloud
  • deepseek-v3.1:671b-cloud
Paste an Ollama API key into Runtime → CTF Web Search and save it to enable live web search without restarting.
